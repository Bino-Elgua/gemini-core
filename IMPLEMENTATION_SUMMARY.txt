INTEGRATION POINTS

autonomousCampaignService.ts ←→ universalAiService.ts
    ↓ Uses for text generation when activeLLM ≠ 'gemini'
    ↓ Routes OpenAI, Claude, Mistral, Groq, DeepSeek


autonomousCampaignService.ts ←→ imageGenerationService.ts
    ↓ Uses for image generation with activeImage provider
    ↓ Routes Stability, DALLE, Leonardo, Black Forest, etc.

    ↓ Reads providers.activeLLM
    ↓ Reads providers.activeImage
    ↓ Both set via Settings UI

    ↓ Gets activeLLM for routing
    ↓ Gets API keys for authentication


imageGenerationService.ts ←→ store.ts
    ↓ Gets activeImage for routing
    ↓ Gets API keys for authentication

ERROR HANDLING & RECOVERY

Image Generation Failure:
┌─────────────────────────────────────────────────────────────────────────┐
│ 1. Try using imageGenerationService with user's selected provider      │
│ 2. Catch any error                                                       │
│ 3. Log warning: "[MEDIA] Image generation failed, using fallback..."    │
│ 4. Fall back to generateCampaignImage (Gemini)                          │
│ 5. Campaign completes successfully                                      │
└─────────────────────────────────────────────────────────────────────────┘

Text Generation Failure:
┌─────────────────────────────────────────────────────────────────────────┐
│ 1. If activeLLM ≠ 'gemini', try universalAiService.generateText()     │
│ 2. universalAiService has its own error handling                        │
│ 3. Falls back to Gemini if provider fails                               │
│ 4. Campaign text generation completes                                   │
